# Frontend Design

> Ship intentional, non-template-looking UI. Fast.

The goal isn't just working UI — it's UI that looks like it was designed, not generated. Two skills enforce this at different layers: taste sets the direction and quality bar, emil executes it.

## Skills

| Skill | Role |
|-------|------|
| [taste](./taste-skill.md) | Design direction, anti-template enforcement, quality criteria |
| [emil](./emil.md) | Senior design engineer persona — implements the taste decisions |

## Anti-Template Policy

These patterns are banned:
- Default card grids with uniform spacing
- Generic hero with centered headline + gradient blob
- Unmodified library defaults (shadcn, Tailwind out-of-box)
- Safe gray-on-white with one accent color

Every meaningful surface should hit at least 4 of: scale contrast, rhythm spacing, depth/layering, real typography, semantic color, designed hover states, editorial composition, texture/atmosphere, purposeful motion, integrated data viz.
